# FitTrack - 减肥饮食记录应用设计文档

## 1. 设计风格选择

### 风格定位：柔和卡片风 (Soft Card UI) + 健康运动配色

**视觉关键词**: 健康、活力、科技、专业、温暖友好

### 2. 核心视觉特征

- **新拟态双层阴影**：卡片采用柔和阴影，营造悬浮感
- **大圆角设计**：borderRadius: 24px，柔和友好
- **渐变按钮**：主色调渐变 + 有色投影
- **图标容器化**：圆形容器包裹图标，统一视觉

### 3. 配色方案

| 角色 | 色值 | 用途 |
|------|------|------|
| Primary (主色) | `#10B981` | Emerald - 积极向上、健康活力 |
| Primary Dark | `#059669` | 深绿 - 强调和交互状态 |
| Background | `#F3F4F6` | 灰白背景 - 通透舒适 |
| Surface | `#FFFFFF` | 卡片背景 - 高对比 |
| Text Primary | `#111827` | 主文字 - 清晰可读 |
| Text Secondary | `#6B7280` | 次要文字 - 辅助信息 |
| Accent Success | `#22C55E` | 成功/达标 |
| Accent Warning | `#F59E0B` | 警告/接近目标 |
| Accent Danger | `#EF4444` | 危险/超标 |

### 4. 字体系统

- **主标题**: 32-40px, Bold, 用于核心数据和标题
- **副标题**: 18-20px, SemiBold, 用于卡片标题
- **正文**: 14-16px, Regular, 用于内容描述
- **辅助文字**: 11-13px, Regular, 用于标签和时间

### 5. 页面结构规划

#### 底部 Tab 导航 (4个Tab)
1. **首页** - 今日饮食记录和热量概览
2. **记录** - 饮食记录添加页面
3. **统计** - 历史数据和趋势分析
4. **我的** - 个人中心和目标设置

#### 页面布局

**首页 (Today)**
- Hero卡片：今日热量环形图 + 营养成分
- 今日饮食时间线：早餐/午餐/晚餐/加餐
- 快速添加按钮

**记录页 (Add)**
- 搜索框：搜索食物
- 食物分类Tab：常见食物/最近/收藏
- 食物列表卡片
- 自定义添加入口

**统计页 (Stats)**
- 时间选择器：周/月视图
- 热量趋势折线图
- 营养成分占比图
- 达标日历

**个人中心 (Profile)**
- 用户信息卡片
- 目标设置入口
- 饮食提醒设置
- 历史记录入口

### 6. 组件设计规范

**卡片组件**
- 背景：白色
- 圆角：24px
- 阴影：`0 4px 12px rgba(16,185,129,0.08)`
- 内边距：20px

**按钮组件**
- 主按钮：渐变背景 + 阴影
- 次按钮：白色背景 + 边框
- 圆角：16px
- 高度：48px

**输入框**
- 背景：#F9FAFB
- 圆角：12px
- 边框：2px transparent
- 聚焦：边框变主色

### 7. 交互动效

- **入场动画**: fadeIn + slideUp, 300ms
- **点击反馈**: scale 0.98, 100ms
- **页面切换**: slide_from_right, 300ms
- **加载状态**: skeleton + shimmer

## 8. 功能模块设计

### 核心功能
1. **饮食记录** - 添加/编辑/删除食物记录
2. **热量追踪** - 实时计算摄入热量
3. **目标设定** - 每日热量目标设置
4. **数据统计** - 历史趋势可视化

### 特色功能
1. **食物数据库** - 常见食物营养数据
2. **快速添加** - 一键记录常用食物
3. **营养分析** - 碳水/蛋白质/脂肪占比
4. **目标对比** - 当日vs目标清晰展示

## 9. 设计禁忌

- 禁止使用过于沉闷的配色
- 禁止使用过小字号导致可读性差
- 禁止卡片之间无间距堆叠
- 禁止纯文字无图标装饰
